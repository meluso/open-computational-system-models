A Catalog of Computational System Models

A browsable list of computational system models — software models of real engineered systems (power grids, robots, spacecraft, chemical plants, and more) that you can actually run: change an input, and the outputs change. Use it to find a model to learn from, build on, or evaluate, whatever your field.

New here? How to read this catalog

What's in here

Each card describes one runnable model or modeling tool. To be listed, a model must be executable: a person can change its inputs or parameters and observe different outputs (a static diagram or spreadsheet with no computation does not qualify). The focus is on models of systems with many interacting parts (roughly 25 or more), grouped into subsystems.

How to use it

  • Search by name, system, inputs, or outputs.
  • Filter by domain (field of application) or by access type (how you get the tool), and sort by how recently each tool was maintained. Reset clears everything.
  • Click a model's title to open its official page. Hover any badge or field label for a one-line explanation.
  • Domains marked "(adjacent)" sit just outside the core focus on engineered systems (for example systems biology or fluid-dynamics solvers) but are included where the model can be run as a multi-part system.

What the labels mean

Each card carries four badges:

Domain
The field or sector the model applies to.
Access type
How you obtain and are allowed to use the tool (see below).
Status
How actively the tool is kept up: Active (under ongoing development), Maintained (kept working with fixes), Dormant (no recent activity but still usable), or Abandoned (no longer maintained).
Confidence
How sure we are that this entry's recorded facts (version, license, scale, and so on) are correct, based on how directly we could confirm them from primary sources. "Last verified" is the date we checked.

Access types, in plain terms:

Permissive open source
Free and open; use, modify, and share with few conditions (e.g. MIT, BSD, Apache).
Weak-copyleft open source
Free and open; changes to the tool's own files stay open, but you may combine it with closed code (e.g. LGPL, MPL).
Copyleft open source
Free and open, but anything you distribute that builds on it must also be released under the same open license (e.g. GPL).
Free, closed source
Free to use, but the source code is not published.
Freemium
A free tier exists; full capability needs a paid plan.
Commercial
A paid product or license.
Campus / employer licensed
Reached through a university or workplace license (e.g. MATLAB/Simulink), not free to the public.

A field shown as unstated — needs direct verification means the source did not state that fact and we chose not to guess.

Every entry records how you can access it, how confident we are in its facts, and the date those facts were last checked. Facts the original source never stated are marked unstated — needs direct verification rather than guessed.

The full dataset, the data format, the longer write-ups, and the log of known gaps are all in the project repository on GitHub.
